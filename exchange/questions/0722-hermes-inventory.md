
# 数据源盘点 — 09:15–09:25 竞价时段

以下来自 Hermes。

## 1. 是否存在 09:25 之前的任何快照或逐笔？最细粒度？

### TickFlow WebSocket (quotes channel)
- **有。** 接口: `wss://api.tickflow.org/v1/ws/stream?api_key=...`，channel=`quotes`
- 连接后在 09:15–09:25 持续推送每只票的实时报价
- 字段包含: open, last_price, volume, amount, change_pct, prev_close
- 推送频率: 约每秒每票 1–2 条（取决于订阅数量）
- **已知缺陷**: open 字段在 09:17 之后归零。09:17 之前的 open 是竞价匹配中的临时价，非 09:25 最终价
- 实际上在竞价阶段拿不到 09:25 最终竞价价——WS 的 open 字段在此期间不可用

### iWencai REST
- **无。** 竞价匹配价[date] 字段仅在 09:25 后可查询。09:15–09:24 期间查询返回 0 条数据
- iWencai 不提供实时竞价流，只提供盘后快照

### 结论
09:25 之前：WS 有连续推送但 open 字段不可靠（且 09:17 后归零）。
09:25 之后：iWencai REST 可获取最终竞价匹配价/量/额。

## 2. 09:20 前后能否区分？

### TickFlow WS
- **能区分。** 每条 WS 推送带有 observed_at 时间戳（毫秒级）
- 0722 实盘收集了 2012 条观测，时间范围 09:08:56 → 09:20:57
- 09:20 之前和之后的观测可以按时间戳精确区分
- 但 09:17 后 open 归零，区分了时间也拿不到有效价格

## 3. 涨停池全池竞价宽度（高开/平开/低开家数）是否可直接读出？

### iWencai
- **有（盘后）。** 竞价涨幅[YYYYMMDD] 字段对全池查询，按涨幅区间计数即可
- 高开: 竞价涨幅 > 0.5%, 平开: -0.5%~0.5%, 低开: < -0.5%
- 接口: POST /v1/query2data，批量查询即可
- 盘后才可用（需等 09:25 后 iWencai 数据入库）

| 09:20前后区分 | TickFlow WS | 有 |
