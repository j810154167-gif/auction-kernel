# 五模块交叉表示例

D1 产出后按此格式呈现。不是排序, 不是评分, 是交叉视图 — 人类扫列头看模式, 扫行看标的。

```
─── 0710 D1 候选池 ───

标的       竞价价  涨幅% │ ①均价 ②量能 ③顺位 │ ④筹码 ⑤反博弈 │ ⑦赔率
─────────────────────────────────────────────────────────
高德红外   14.64  +9.99 │  ✗     ⚠     ?   │   ?     ?    │ 不对称
崇达技术   18.87 +10.03 │  ✗     ⚠     ?   │   ?     ?    │ 不对称
中京电子   18.29  +1.27 │  ✓     ⚠     ?   │   ?     ?    │ 平衡
通富微电   75.97  +5.27 │  ✓     ⚠     ?   │   ?     ?    │ 不对称

✓=通过  ✗=失败  ⚠=预警  ?=未知(数据未接入)
⑦赔率: 不对称=上行空间<下行空间×0.3
```

**对照组定义:**

| 模块 | 对照组 | 数据源 |
|------|--------|--------|
| ①均价 | 竞价价 vs [VWAP底, VWAP×天花板] | TickFlow klines |
| ②量能 | 竞价量/额 vs 前日竞价量/额 | TickFlow WS (REST无竞价量→全量⚠) |
| ③顺位 | 个股涨幅 vs 板块涨幅 | iWencai板块涨幅API |
| ④筹码 | 买卖盘供需比 (总买/总卖) | TickFlow WS depth |
| ⑤反博弈 | 竞价最后60s轨迹 + 开盘首笔量 | TickFlow WS quotes流 |
| ⑦赔率 | 上行空间(天花板-竞价价) vs 下行空间(竞价价-VWAP底) | K线+竞价价 |

**人类动作:**
1. 先扫列头 — 哪个列 ✗ 最多? 哪个列全 ? (数据盲区)? 5秒
2. 再扫行 — 哪只标的多列同时 ✗? 哪只全 ✓ 但⑦不对称? 10秒
3. 聚焦 — 最多 5只需要细看, 其余折叠
4. 决策 — 不出首选, 不出排名, 不出评分
