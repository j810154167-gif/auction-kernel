# Jackson A股竞价项目：新会话交互路由

## 触发时机

本文件在新会话加载。会话尚未确定能力模式时，按“首轮判路”处理用户第一条有效消息。

模式只决定加载什么、允许做什么、调用哪个入口；不得借模式切换修改交易公式、排序或冻结输出。

## 首轮判路

1. 用户明确输入某个模式名，直接进入该模式，不再显示菜单。
2. 用户首条消息含唯一归属的点火词，直接进入对应模式并处理该请求。
3. 用户请求明显属于运维能力，直接进入运维模式。
4. 无法确定模式时，必须调用交互选择工具，提供且只提供以下四个选项：
   - 运维 / 接力
   - 运行 / 进攻
   - 运行 / 防守
   - 极限 / 博弈
5. 若当前界面不支持交互选择，则显示同样四个文本选项，请用户回复 `运维`、`进攻`、`防守`或`极限`。

不得把选项枚举写进提问正文；使用交互选择工具时，各选项必须放入工具的 choices 数组。

## 四模式

### 运维 / 接力

**直达词：** `运维`、`接力`、`故障`、`恢复`、`核准`、`定版`、`运行追溯`、`版本核对`

**进入后加载：**
- `auction-operations`
- 不默认加载两份进攻策略 Skill；只按当前故障对象定向读取相关引擎、产物、会话和操作约定
- 涉及防守运行时遵守防守壳边界，按需加载 `constraint-defense`

**能力路由：**
- 运行追溯 → 以指定会话原文和真实工具输出重建时间线
- 故障定位 → 判断失败节点、下一窗口影响和当前最大阻断
- 运行恢复 → 处理数据、环境、契约、引擎或提示栈故障并实跑恢复链
- 版本核对 → 对照当前代码、配置、handoff、版本链及执行证据
- 定版交接 → 固定已验证版本，只向运行模式交付入口、就绪状态、产物和停止条件
- 任务维护 → 保留原始问题台账，以端到端运行切片组织实施，不让看板代替运行恢复

**工作准则：**
- 每次先回答：哪个节点失败、影响什么、最大阻断是什么、如何恢复、怎样实跑证明
- 不得把权限、确认、测试路径、看板或审批流程当成运维任务主体；对应动作发生前再检查边界
- 生产改动只执行 Jackson 已确认的具体范围；不得自主增加功能、互验层、状态机或运维平台
- D1评分、排序和冻结8列输出不在运维中自主修改
- 运维完成以真实恢复结果为准，不以提案、流程或文档完成为准

### 运行 / 进攻

**直达词：** `进攻`、`scan`、`风控`、`启动`、`auction`、`D1`、`延`、`T`

**进入后加载：**
- `morning-auction-strategy`
- 不默认加载 `scoring-strategy` 全文；评分和策略背景只在非运行窗口按需读取

**运行目录：** `/Users/fiona/.hermes/workspace`

**功能路由：**
- `scan YYYY-MM-DD` → `python3 scripts/auction_engine.py scan YYYY-MM-DD`
- `风控 YYYY-MM-DD` → `python3 scripts/auction_engine.py 风控 YYYY-MM-DD`
- `review YYYY-MM-DD` → `python3 scripts/auction_engine.py review YYYY-MM-DD`
- `review_ready YYYY-MM-DD` / `复盘 YYYY-MM-DD` → `python3 scripts/auction_engine.py review_ready YYYY-MM-DD`
- `启动` → `python3 scripts/auction_engine.py 启动`
- `auction` → `python3 scripts/auction_engine.py auction`
- `D1` → `python3 scripts/auction_engine.py D1`
- `延` → `python3 scripts/auction_engine.py 延`
- `T` → `python3 scripts/auction_engine.py T`

**盘后完整链：** `scan 日期 → 风控 日期 → review 日期 → review_ready 日期`。`review`只生成分钟K复盘缓存，不等于盘后READY；最后必须执行`review_ready`汇总验收。

**硬门禁：**
- 只执行点火词对应节点，不自主探索
- 节点失败时只报告：节点、事实、影响、停止状态；随后停止，不读源码、不补环境、不改生产、不人工改写正式产物
- 故障调查与恢复转入运维模式；运维交回固定入口和READY/BLOCKED状态后，进攻模式才继续
- 不临场优化公式或排序
- D1 stdout 原样渲染，不包装、不翻译、不重排、不增删列
- 排名归引擎，剃刀归人

### 运行 / 防守

**直达词：** `防守`、`建区间`、`监控`、`查防守`、`强化`

**进入后加载：**
- `constraint-defense`

**运行目录：** `/Users/fiona/.hermes/workspace/profiles-defense`

**功能路由：**
- `建区间 YYYY-MM-DD` → `python3 scripts/defense_engine.py 建区间 YYYY-MM-DD`
- `监控` → `python3 scripts/defense_engine.py 监控`
- `查防守 SYM` → `python3 scripts/defense_engine.py 查防守 SYM`
- `强化 SYM [成本]` → `python3 scripts/defense_engine.py 强化 SYM [成本]`
- 维护台账 → `config/ledger.csv`
- 维护 L1 → `config/l1_map.json`

**硬门禁：**
- 只访问防守壳的 `core/`、`config/`、`handoff/`、`memories/`及其执行入口
- 不读取、不调用进攻壳
- 机器不自动推断 L1
- 只报事实，不决策、不追问、不报平安
- 沉默是正常；L2 触发才报警
- 每天都是新的买入

### 极限 / 博弈

**直达词：** `极限`、`极限模式`、`纯博弈`、`迷宫`、`奖励记忆`、`错误降级`

**进入后加载：**
- `/Users/fiona/.hermes/workspace/极限模式工作区/MODE.md`
- 不按语义自动搜索或加载其他 Skill

**工作目录：** `/Users/fiona/.hermes/workspace/极限模式工作区`

**能力路由：**
- `迷宫：原文` → 逐字追加到 `memories/original_maze.md`
- `认知：内容` → 在 `memories/cognition_ledger.jsonl` 建立 `candidate`
- `奖励 ID 证据` → 经 Jackson 确认后上调一级
- `错误 ID 证据` → 经 Jackson 确认后下调一级
- `隔离 ID 证据` → 经 Jackson 确认后转为 `quarantined`
- `查看认知` → 读取本模式认知账本
- 其他市场问题 → 纯金融市场博弈流

**硬门禁：**
- 这是第四种交互模式，不是第四套引擎
- 只围绕错价、叙事差、对手盘、赔率和超额收益
- 不自动调用进攻/防守引擎，不修改公式、排序或冻结输出
- 不把市场问题转成工程项目；数据缺口只报告对当前判断的影响
- 不自动加载非项目 Skill；不复述用户语言冒充理解
- 认知升降级必须由 Jackson 明示确认；不得自动回灌其他三模式

## 歧义词边界

- `风控`只属于进攻壳的盘后 `risk_scan`
- `监控`只属于防守壳
- 不得按自然语言近义词互换二者

## 会话内切换

只有明确指令才能切换：

- `切到运维`
- `切到进攻`
- `切到防守`
- `切到极限`

切换后只回复一行：`已切换到「××模式」；可用功能：……`

不因一句跨域讨论自动切换。若当前模式收到另一模式的执行点火词，先说明目标模式并请求一次确认；确认后切换并执行，避免双壳混跑。

## 四模式工作区隔离

四模式的测试工作区必须隔离，禁止跨模式共用：

- 运维历史与任务根：`/Users/fiona/.hermes/workspace/运维模式工作区`（保留既有目录名以维持证据路径，不代表模式仍为治理）
- 进攻测试根：`/Users/fiona/.hermes/workspace/进攻模式工作区/过程垃圾站`
- 防守测试：每次由用户指定防守壳内独立工作区；不得使用治理或进攻测试根。
- 极限模式独立工作区：`/Users/fiona/.hermes/workspace/极限模式工作区`；其记忆不得自动回灌其他模式。

任何会创建、修改文件或产生过程产物的测试，只能进入对应模式的隔离工作区。运维开始实施已确认任务时，使用该任务已经指定的工作区；确实没有可用路径时才询问一次，不得反复把路径确认当成任务主体。只读调查可直接进行；过程文档属于低置信度材料，不得直接作为生产事实、定版依据或回灌生产。

正式交接可保留在交接仓库根部；非实盘、非定版过程文档只能进入对应测试子工作区。生产引擎和运行模式Agent禁止自动读取历史过程材料、进攻“过程垃圾站”或防守测试工作区；运维接力按当前任务定向读取，不回灌完整历史。
