# 进攻模式内核归零重构

## 当前原点

新内核不复原0618旧实现，也不继承0722生产内核。已有代码只能作为可取用的数据能力材料。

核心只有两段和一条消费链：

```text
T-1 review内核
→ 产出每只候选股的自身基线、空间坐标与题材事实
→ T日09:15—09:25竞价内核持续保存量价时间轨迹
→ D1把T日行为与T-1基线逐票对照，并呈现题材/板块位置
→ 机器压缩，Jackson剃刀
```

## 认识内核

```text
价格是结果
量是承诺
时间是证词
空间是位置
题材是原因
```

五者不是五个加权因子，也不是五道闸门。禁止建立“量分、价分、时间分、空间分、题材分”的总分公式；禁止把任何维度转换成通过/不通过、保留/淘汰、资格条件、罚分或固定阈值。

新内核观察的是：一只股票在T日形成了什么竞价行为；机器把它自己的T-1真实交易基线、竞价形成过程、市场位置和题材事实并排保真呈现，不替Jackson裁定这些关系是否“合格”或“可解释”。

## 两个核心

### 一、T-1 review

review不是生成泛化标签，也不是扫描十日分钟量后判定“机械刷/僵尸量”。它负责为次日竞价准备可直接消费的事实基线：

- 候选池身份与名称；
- T-1收盘价、成交量、成交额；
- T-1全天VWAP、最低价、最高价；
- T-1成交密集价格区；
- T-1分时成交路径；
- T-1涨停形成时间、开板次数、封单等客观事实；
- 题材原因、行业与同题材成员；
- 每个字段的来源、日期和可用状态。

review产物必须按symbol组织，使T日可以逐票直接对照。

### 二、T日竞价

竞价内核保存09:15—09:25全过程，而不是用`latest[symbol]`覆盖历史。每条观测至少保留：

- symbol；
- 交易所或源时间戳；
- 接收时间戳；
- 匹配价/竞价价；
- 匹配量；
- 匹配额；
- 昨收；
- 数据来源。

竞价完成后，从原始轨迹形成事实，而不是结论：

- 09:25最终竞价价、量、额；
- 价格路径；
- 量额增长路径；
- 09:20前后变化；
- 撤退、回补和末段突击的客观差值；
- 同题材成员在相同时间轴上的同步情况。

## 消费连接

D1必须读取两类输入：

```text
review/T-1.json
auction/T/trajectory.jsonl + close.json
```

D1的每一只输出记录都必须包含`consumption`证据：

- review日期；
- review基线字段；
- T日竞价轨迹观测数；
- 最早/最晚观测时间；
- 对照所使用的字段；
- 题材与同题材成员事实是否可用。

若没有T-1 review，不能悄悄退化成只看价格；若没有T日轨迹，不能把最终快照冒充时间维度。

## 排序与裁决

机器必须压缩候选，但本阶段不预设新排序公式。第一阶段先产出逐票完整证据对象，观察哪些事实在历史回放中有区分度，再由Jackson确认排序原则。

机器不做：

- 五维加权总分；
- 任何通过/不通过闸门；
- 任何固定阈值淘汰；
- 任何维度罚分或资格条件；
- 题材质量自动裁决；
- 用板块字符串冒充题材理解；
- 用最终快照冒充竞价过程；
- 用文件存在冒充已经被D1消费。
